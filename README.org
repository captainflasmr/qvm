#+title: qemu-manager.el --- QEMU VM manager for Emacs
#+author: James Dyer
#+version: 1.1.0

*qemu-manager.el* manages QEMU virtual machines directly from Emacs. It calls ~qemu-system-x86_64~, ~qemu-img~, ~vncviewer~, ~spicy~, ~ssh~, and ~rsync~ directly -- no external wrapper script required.

* Features

- *VM List Buffer*: A tabulated list (~qemu-manager-list~) showing all your VMs with their status (running/stopped), memory, CPUs, disk usage, SSH port, and display type.
- *Transient Menu*: Press ~?~ in the list buffer (or ~M-x qemu-manager-menu~) for a discoverable menu of all commands, grouped into Lifecycle, Connect, Snapshots, and Tools.
- *Async Operations*: VM operations (start, stop, etc.) run asynchronously, and their output is shown in a dedicated ~*qemu-manager output*~ buffer.
- *TRAMP Integration*: Quickly open /dired/ or an /eshell/ on a running VM via SSH (localhost tunneling).
- *Display Support*: Launch VNC or SPICE viewers for any running VM. SPICE provides clipboard sharing, dynamic resolution, and better keyboard input handling.
- *File Transfer*: Easily rsync files (or dired marked files) to a VM.
- *Shared Folders*: Share host directories into a VM via virtiofs. qemu-manager launches and supervises ~virtiofsd~ alongside the VM automatically.
- *Clipboard Sharing*: Copy/paste between host and VM over SSH (~w~ / ~y~), with auto-detection of guest clipboard tools (wl-clipboard, xclip, xsel). Works with both VNC and SPICE.
- *Keyboard Setup*: Configure keyboard remaps (CapsLock/Right Alt → Ctrl) and sticky keys on GNOME VM guests via SSH.
- *Display Switching*: Toggle between VNC and SPICE display types without recreating the VM.
- *Snapshots*: Create, list, restore, and delete qcow2 internal snapshots on stopped VMs, with completing-read for snapshot selection.
- *Cloning*: Clone VMs as a full independent copy or a linked (COW) clone that shares the source disk for speed and space efficiency.
- *Creation*: Create new VMs from ISO images directly from Emacs, prompting for disk size, memory, CPU count, and display type.

* Requirements

- *Emacs 28.1* or newer
- *transient* 0.4.0+ (bundled with Emacs 29+, or available from MELPA)
- Built-in Emacs libraries: ~tabulated-list~, ~tramp~

External tools (must be in ~PATH~):

| Tool                   | Used for                                 |

* Installation

** Manual

Download ~qemu-manager.el~ to your load path and add the following to your ~init.el~:

#+begin_src elisp
(require 'qemu-manager)
(setq qemu-manager-base-dir "~/VM") ; adjust to your VM storage directory
#+end_src

** Using use-package

#+begin_src elisp
(use-package qemu-manager
  :load-path "~/source/repos/qemu-manager"
  :bind ("C-c v" . qemu-manager-list)
  :config
  (setq qemu-manager-base-dir "~/VM"))
#+end_src

* Usage

Run ~M-x qemu-manager-list~ to open the VM manager buffer, then press ~?~ to open the transient menu. You can also run ~M-x qemu-manager-menu~ from anywhere -- outside the list buffer it offers a minimal set of commands that prompt for a VM name.

** Snapshots

Create, list, restore, and delete internal qcow2 snapshots on stopped VMs.

In the Emacs list buffer, ~+~ creates a snapshot and ~R~ / ~X~ restore / delete with completing-read over existing tags. ~N~ lists all snapshots for the VM at point.

** Cloning

Clone a stopped VM as either a full independent copy or a linked (copy-on-write) clone. Press ~C~ in the list buffer.

Full clones are independent but take time and disk space proportional to the source disk. Linked clones are instant and space-efficient, but depend on the source disk -- if the source is deleted or corrupted, the linked clone breaks.

New ports (SSH, VNC, SPICE) are auto-assigned to avoid collisions.

** SPICE Display

SPICE offers advantages over VNC for Wayland hosts (e.g. SwayWM) where host keyboard remappings do not pass through to the guest:

- Clipboard sharing between host and guest (via ~spice-vdagent~)
- Dynamic resolution resizing
- Better input handling

Toggle an existing VM between display types with ~D~ in the list buffer, or:

#+begin_src elisp
(qemu-manager-display "myvm") ; toggles between vnc and spice
#+end_src

Existing VMs default to VNC for backward compatibility.

*** SwayWM: Keyboard Shortcut Passthrough

By default, the ~spicy~ SPICE client requests keyboard shortcut inhibition from the Wayland compositor, causing host keybindings (e.g. ~Super+u~ to switch desktops) to be sent to the guest instead of being handled by sway. To prevent this, add the following to your sway config:

#+begin_src conf :tangle no
for_window [app_id="spicy"] shortcuts_inhibitor disable
#+end_src

Reload sway (~swaymsg reload~) and restart the SPICE viewer for the change to take effect.

** Keyboard Setup for VM Guests

Host XKB settings (sticky keys, key remaps) do not pass through SPICE or VNC to the guest since keyboard input is sent as raw scancodes. The keyboard command configures these inside the guest via SSH (~k~ in list buffer):

This applies to GNOME Wayland guests:

- GB keyboard layout
- CapsLock → Ctrl (via ~ctrl:nocaps~)
- Right Alt → Ctrl (via ~ctrl:ralt_rctrl~)
- Sticky keys enabled

Log out and back in to the guest for changes to take effect.

** Shared Folders (virtiofs)

Share a host directory into a VM via virtiofs. Press ~F~ in the list buffer (or ~M-x qemu-manager-share-set~) to set a host path and mount tag, then start the VM. An empty path clears the share.

When a share is configured, ~qemu-manager-start~ launches ~virtiofsd~ on a Unix socket in the VM directory and adds the matching ~memory-backend-memfd~, ~vhost-user-fs-pci~, and chardev args to QEMU. The daemon is terminated automatically when the VM exits.

Mount the share from inside the guest (using the tag you chose):

#+begin_src sh :tangle no
sudo mount -t virtiofs hostshare /mnt/host
#+end_src

Or add it to ~/etc/fstab~:

#+begin_src conf :tangle no
hostshare  /mnt/host  virtiofs  defaults  0  0
#+end_src

Requires a Linux guest with ~virtiofs~ kernel support (5.4+) and ~virtiofsd~ on the host. qemu-manager looks for the daemon in ~PATH~ and then falls back to ~/usr/lib/virtiofsd~ (Arch), ~/usr/libexec/virtiofsd~ (Fedora/RHEL), and ~/usr/lib/qemu/virtiofsd~; set ~qemu-manager-virtiofsd-binary~ if yours lives elsewhere.

Virtiofsd is launched with ~--sandbox=none~ by default so it can run as a regular user -- the ~namespace~ sandbox mode needs ~CAP_SYS_ADMIN~. If you run qemu-manager as root (or with the right capabilities), remove it from ~qemu-manager-virtiofsd-args~ for stronger isolation.

** Clipboard Sharing (VNC)

SPICE provides native clipboard sharing via ~spice-vdagent~. For VNC mode (or as a fallback), clipboard can be shared over SSH:

- ~w~ in the list buffer copies VM clipboard → host (kill ring + system clipboard)
- ~y~ pastes host clipboard → VM
- ~I~ installs clipboard tools on the guest (auto-detects package manager)

The copy and paste commands auto-detect which clipboard tool is available on the guest (~wl-paste~ / ~wl-copy~ for Wayland, ~xclip~ or ~xsel~ for X11).

Requires ~wl-copy~ / ~wl-paste~ on the host (included with sway).

* Customization

| Variable                  | Description                                 | Default                      |
|---------------------------+---------------------------------------------+------------------------------|
| ~qemu-manager-base-dir~            | Directory where VMs are stored              | =~/VM=                       |
| ~qemu-manager-qemu-binary~         | Path to QEMU system emulator                | =qemu-system-x86_64= in PATH |
| ~qemu-manager-vnc-viewer~          | VNC viewer program                          | =vncviewer= in PATH          |
| ~qemu-manager-vnc-viewer-args~     | Extra args for VNC viewer                   | =("-RemoteResize=1")=        |
| ~qemu-manager-spice-viewer~        | SPICE viewer program                        | =spicy= in PATH              |
| ~qemu-manager-virtiofsd-binary~    | Path to virtiofsd daemon                    | =virtiofsd= / =/usr/libexec/virtiofsd= |
| ~qemu-manager-virtiofsd-args~      | Extra args for virtiofsd                    | =("--cache=auto")=           |
| ~qemu-manager-default-memory~      | Default memory for new VMs                  | ="4G"=                       |
| ~qemu-manager-default-cpus~        | Default CPU count for new VMs               | ="4"=                        |
| ~qemu-manager-default-disk~        | Default disk size for new VMs               | ="40G"=                      |
| ~qemu-manager-default-ssh-port~    | Starting SSH port for auto-assignment       | =2222=                       |
| ~qemu-manager-default-vnc-display~ | Starting VNC display for auto-assignment    | =1=                          |
| ~qemu-manager-default-spice-port~  | Starting SPICE port for auto-assignment     | =5930=                       |
| ~qemu-manager-default-display~     | Default display type (="vnc"= or ="spice"=) | ="vnc"=                      |
| ~qemu-manager-default-user~        | Default SSH username for new VMs            | Current login name           |
| ~qemu-manager-output-buffer~       | Buffer name for command output              | ="*qemu-manager output*"=             |
