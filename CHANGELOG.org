#+title: Changelog
#+author: James Dyer

* 1.1.0 — <2026-04-13>

** Added

- Virtiofs shared folders: set ~SHARE_PATH~ / ~SHARE_TAG~ on a VM (via ~qemu-manager-share-set~ or ~F~ in the list buffer) and ~qemu-manager-start~ launches ~virtiofsd~ alongside QEMU on a per-VM Unix socket, wires up ~memory-backend-memfd~ + ~vhost-user-fs-pci~ args, and tears the daemon down when the VM exits
- ~qemu-manager-virtiofsd-binary~ and ~qemu-manager-virtiofsd-args~ customizations
- ~qemu-manager-share-set~ interactive command and ~qemu-manager-list-share-set~ (~F~) list-buffer wrapper; share entry added to the Tools group in the transient menu
- ~qemu-manager-info~ now reports the configured share path and mount tag

** Fixed

- ~qemu-manager-eshell~ no longer errors with "Defining as dynamic an already lexical var: eshell-buffer-name" on recent Emacs; the eshell buffer is now created via ~(eshell t)~ and renamed after the fact
- ~qemu-manager-snapshot-list~ and ~qemu-manager--snapshot-tags~ pass ~-U~ to ~qemu-img snapshot -l~ so listing snapshots works while the VM is running and holds the qcow2 write lock

** Changed

- ~qemu-manager-menu~ transient now displays all four groups (Lifecycle, Connect, Snapshots, Tools) in a single row

* 1.0.0 — <2026-04-05>

Pure Elisp rewrite -- removed dependency on the =qemu-manager= bash script.

** Changed

- ~qemu-manager.el~ now calls QEMU tools (~qemu-system-x86_64~, ~qemu-img~), SSH utilities, and display viewers directly -- the external ~qemu-manager~ shell script is no longer required
- ~qemu-manager-start~ builds QEMU command-line arguments in Elisp and launches via ~make-process~ with PID file management and automatic cleanup on exit
- ~qemu-manager-stop~ uses ~signal-process~ (SIGTERM) with timer-based polling, escalating to SIGKILL after 10 seconds
- ~qemu-manager-info~ now renders VM details purely in Elisp (no external process)
- ~qemu-manager-display~ toggles display type by updating ~vm.conf~ directly (no external process)
- ~qemu-manager-create~ chains ~qemu-img create~ -> config write -> QEMU ISO boot via async callbacks
- ~qemu-manager-clone~ calls ~qemu-img create~ (linked) or ~cp~ (full) directly, with auto-assigned ports
- ~qemu-manager-snapshot-*~ commands call ~qemu-img snapshot~ directly
- ~qemu-manager-vnc~ / ~qemu-manager-spice~ launch viewer programs directly
- ~qemu-manager-keyboard~ runs gsettings commands via a single SSH call
- ~qemu-manager-clip-copy~ / ~qemu-manager-clip-paste~ use synchronous SSH + ~wl-copy~ / ~wl-paste~
- ~qemu-manager-ssh-copy-id~ runs ~ssh-copy-id~ directly in a term buffer; a sentinel handles permission fixing and SSH config entry creation
- Clip install runs ~ssh -t~ directly in a term buffer with package manager detection

** Added

- ~qemu-manager-qemu-binary~ customization (path to ~qemu-system-x86_64~)
- ~qemu-manager-vnc-viewer~ / ~qemu-manager-vnc-viewer-args~ / ~qemu-manager-spice-viewer~ customizations for viewer programs
- ~qemu-manager-default-memory~ / ~qemu-manager-default-cpus~ / ~qemu-manager-default-disk~ customizations for VM creation defaults
- ~qemu-manager-default-ssh-port~ / ~qemu-manager-default-vnc-display~ / ~qemu-manager-default-spice-port~ for port auto-assignment starting values
- ~qemu-manager-default-display~ / ~qemu-manager-default-user~ customizations
- Port allocation helpers (~qemu-manager--next-free-port~, ~qemu-manager--next-free-vnc~) using ~ss~ to detect used ports
- Config writing (~qemu-manager--write-conf~, ~qemu-manager--update-conf-value~) for creating and modifying ~vm.conf~ from Elisp
- QEMU process launcher (~qemu-manager--start-qemu~) with PID file lifecycle, sentinel cleanup, and automatic list refresh on VM exit

** Removed

- ~qemu-manager-executable~ customization variable
- ~qemu-manager--run-command~ function (replaced by ~qemu-manager--run-process~ which takes a program + args directly)
- Dependency on the external ~qemu-manager~ shell script

* 0.5.0 — <2026-04-05>

Transient menu for discoverable command access.

** Added

- ~qemu-manager-menu~ transient prefix command with 2×2 grouped layout: Lifecycle, Connect, Snapshots, Tools
- ~?~ keybinding in list buffer to invoke the transient menu
- Context-aware menu: shows list-buffer commands when in ~*qemu-manager*~, standalone commands elsewhere
- ~transient~ added to ~Package-Requires~

** Changed

- ~n~ / ~p~ now navigate the list (next/previous line)
- Snapshot create moved from ~n~ to ~+~
- Removed minibuffer keybinding hint on ~qemu-manager-list~ startup (replaced by transient menu)

* 0.4.0 — <2026-04-04>

Snapshots and VM cloning.

** Added

- ~qemu-manager snapshot <name> create|list|restore|delete [tag]~ for managing qcow2 internal snapshots on stopped VMs
- ~qemu-manager clone <source> <new-name> [--linked]~ to clone a VM
  - Full clone: independent copy of the entire disk
  - Linked clone (~--linked~): copy-on-write overlay backed by the source disk (fast, space-efficient)
  - Auto-assigns new SSH, VNC, and SPICE ports to avoid collisions
- Emacs: ~qemu-manager-snapshot-create~, ~qemu-manager-snapshot-list~, ~qemu-manager-snapshot-restore~, ~qemu-manager-snapshot-delete~ interactive commands
- Emacs: ~qemu-manager-clone~ interactive command with prompt for full vs linked clone
- Emacs: ~n~ (snapshot create), ~N~ (snapshot list), ~R~ (snapshot restore), ~X~ (snapshot delete), ~C~ (clone) keybindings in list buffer
- Emacs: ~qemu-manager--snapshot-tags~ helper parses ~qemu-img snapshot -l~ output for completing-read on restore and delete

* 0.3.0 — <2026-04-03>

Clipboard sharing over SSH, display switching, and SwayWM integration notes.

** Added

- ~qemu-manager clip <name> copy|paste~ for clipboard sharing over SSH between host and VM
  - Auto-detects guest clipboard tool: ~wl-paste~/~wl-copy~ (Wayland), ~xclip~ (X11), ~xsel~
  - Single SSH call per operation (no double password prompts)
- ~qemu-manager clip <name> install~ to install clipboard tools (~wl-clipboard~ + ~xclip~) on the guest
  - Auto-detects package manager (zypper, apt, dnf, pacman, apk)
  - Kills PackageKit if it is blocking the package manager
  - Runs in a terminal buffer from Emacs for interactive sudo
- ~qemu-manager display <name> [vnc|spice]~ to show or switch display type without recreating the VM
- Emacs: ~w~ (clip copy), ~y~ (clip paste), ~I~ (clip install), ~D~ (toggle display) keybindings in list buffer
- Documented SwayWM ~shortcuts_inhibitor~ workaround for SPICE keyboard grab

* 0.2.0 — <2026-04-03>

SPICE display support, keyboard setup command, and Emacs UI updates.

** Added

- SPICE display backend as an alternative to VNC (~--display spice~ on ~qemu-manager create~)
  - Uses QXL graphics with ~spice-vdagent~ for clipboard sharing and dynamic resolution
  - ~qemu-manager spice <name>~ command to open SPICE viewer (~spicy~)
  - ~qemu-manager run~ auto-selects the correct viewer based on VM display type
- ~qemu-manager keyboard <name>~ command to configure keyboard remaps and sticky keys on GNOME Wayland guests via SSH
  - GB layout, CapsLock → Ctrl, Right Alt → Ctrl, sticky keys
- Emacs: ~qemu-manager-spice~ command and ~V~ keybinding in list buffer
- Emacs: ~qemu-manager-keyboard~ command and ~k~ keybinding in list buffer
- ~VM_DISPLAY~ and ~VM_SPICE_PORT~ fields in ~vm.conf~

** Changed

- Emacs: list buffer columns updated from "VNC" to "Display" and "Port"
- ~qemu-manager list~ and ~qemu-manager info~ now show display type and corresponding port
- Bumped ~qemu-manager.el~ version to 0.2.0

** Backward Compatibility

- Existing VMs without ~VM_DISPLAY~ in their config default to ~vnc~
- No changes required to existing VM configurations

* 0.1.0 — <2026-04-01>

Initial release.

** Added

- ~qemu-manager~ shell script: Docker-inspired CLI for managing QEMU VMs
  - create, install, start, run, stop, ssh, scp, vnc, list, info, rename, rm
  - VNC display with auto-assigned ports
  - User-mode networking with SSH port forwarding
- ~qemu-manager.el~ Emacs interface
  - Tabulated list buffer with keybindings for all operations
  - TRAMP integration for dired and eshell
  - Async command output buffer
  - VM creation from ISO with dired integration
