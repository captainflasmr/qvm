#+author: James Dyer

* 1.0.0 — <2026-04-05>

Pure Elisp rewrite -- removed dependency on the =qvm= bash script.

** Changed

- ~qvm.el~ now calls QEMU tools (~qemu-system-x86_64~, ~qemu-img~), SSH utilities, and display viewers directly -- the external ~qvm~ shell script is no longer required
- ~qvm-start~ builds QEMU command-line arguments in Elisp and launches via ~make-process~ with PID file management and automatic cleanup on exit
- ~qvm-stop~ uses ~signal-process~ (SIGTERM) with timer-based polling, escalating to SIGKILL after 10 seconds
- ~qvm-info~ now renders VM details purely in Elisp (no external process)
- ~qvm-display~ toggles display type by updating ~vm.conf~ directly (no external process)
- ~qvm-create~ chains ~qemu-img create~ -> config write -> QEMU ISO boot via async callbacks
- ~qvm-clone~ calls ~qemu-img create~ (linked) or ~cp~ (full) directly, with auto-assigned ports
- ~qvm-snapshot-*~ commands call ~qemu-img snapshot~ directly
- ~qvm-vnc~ / ~qvm-spice~ launch viewer programs directly
- ~qvm-keyboard~ runs gsettings commands via a single SSH call
- ~qvm-clip-copy~ / ~qvm-clip-paste~ use synchronous SSH + ~wl-copy~ / ~wl-paste~
- ~qvm-ssh-copy-id~ runs ~ssh-copy-id~ directly in a term buffer; a sentinel handles permission fixing and SSH config entry creation
- Clip install runs ~ssh -t~ directly in a term buffer with package manager detection

** Added

- ~qvm-qemu-binary~ customization (path to ~qemu-system-x86_64~)
- ~qvm-vnc-viewer~ / ~qvm-vnc-viewer-args~ / ~qvm-spice-viewer~ customizations for viewer programs
- ~qvm-default-memory~ / ~qvm-default-cpus~ / ~qvm-default-disk~ customizations for VM creation defaults
- ~qvm-default-ssh-port~ / ~qvm-default-vnc-display~ / ~qvm-default-spice-port~ for port auto-assignment starting values
- ~qvm-default-display~ / ~qvm-default-user~ customizations
- Port allocation helpers (~qvm--next-free-port~, ~qvm--next-free-vnc~) using ~ss~ to detect used ports
- Config writing (~qvm--write-conf~, ~qvm--update-conf-value~) for creating and modifying ~vm.conf~ from Elisp
- QEMU process launcher (~qvm--start-qemu~) with PID file lifecycle, sentinel cleanup, and automatic list refresh on VM exit

** Removed

- ~qvm-executable~ customization variable
- ~qvm--run-command~ function (replaced by ~qvm--run-process~ which takes a program + args directly)
- Dependency on the external ~qvm~ shell script

* 0.5.0 — <2026-04-05>

Transient menu for discoverable command access.

** Added

- ~qvm-menu~ transient prefix command with 2×2 grouped layout: Lifecycle, Connect, Snapshots, Tools
- ~?~ keybinding in list buffer to invoke the transient menu
- Context-aware menu: shows list-buffer commands when in ~*qvm*~, standalone commands elsewhere
- ~transient~ added to ~Package-Requires~

** Changed

- ~n~ / ~p~ now navigate the list (next/previous line)
- Snapshot create moved from ~n~ to ~+~
- Removed minibuffer keybinding hint on ~qvm-list~ startup (replaced by transient menu)

* 0.4.0 — <2026-04-04>

Snapshots and VM cloning.

** Added

- ~qvm snapshot <name> create|list|restore|delete [tag]~ for managing qcow2 internal snapshots on stopped VMs
- ~qvm clone <source> <new-name> [--linked]~ to clone a VM
  - Full clone: independent copy of the entire disk
  - Linked clone (~--linked~): copy-on-write overlay backed by the source disk (fast, space-efficient)
  - Auto-assigns new SSH, VNC, and SPICE ports to avoid collisions
- Emacs: ~qvm-snapshot-create~, ~qvm-snapshot-list~, ~qvm-snapshot-restore~, ~qvm-snapshot-delete~ interactive commands
- Emacs: ~qvm-clone~ interactive command with prompt for full vs linked clone
- Emacs: ~n~ (snapshot create), ~N~ (snapshot list), ~R~ (snapshot restore), ~X~ (snapshot delete), ~C~ (clone) keybindings in list buffer
- Emacs: ~qvm--snapshot-tags~ helper parses ~qemu-img snapshot -l~ output for completing-read on restore and delete

* 0.3.0 — <2026-04-03>

Clipboard sharing over SSH, display switching, and SwayWM integration notes.

** Added

- ~qvm clip <name> copy|paste~ for clipboard sharing over SSH between host and VM
  - Auto-detects guest clipboard tool: ~wl-paste~/~wl-copy~ (Wayland), ~xclip~ (X11), ~xsel~
  - Single SSH call per operation (no double password prompts)
- ~qvm clip <name> install~ to install clipboard tools (~wl-clipboard~ + ~xclip~) on the guest
  - Auto-detects package manager (zypper, apt, dnf, pacman, apk)
  - Kills PackageKit if it is blocking the package manager
  - Runs in a terminal buffer from Emacs for interactive sudo
- ~qvm display <name> [vnc|spice]~ to show or switch display type without recreating the VM
- Emacs: ~w~ (clip copy), ~y~ (clip paste), ~I~ (clip install), ~D~ (toggle display) keybindings in list buffer
- Documented SwayWM ~shortcuts_inhibitor~ workaround for SPICE keyboard grab

* 0.2.0 — <2026-04-03>

SPICE display support, keyboard setup command, and Emacs UI updates.

** Added

- SPICE display backend as an alternative to VNC (~--display spice~ on ~qvm create~)
  - Uses QXL graphics with ~spice-vdagent~ for clipboard sharing and dynamic resolution
  - ~qvm spice <name>~ command to open SPICE viewer (~spicy~)
  - ~qvm run~ auto-selects the correct viewer based on VM display type
- ~qvm keyboard <name>~ command to configure keyboard remaps and sticky keys on GNOME Wayland guests via SSH
  - GB layout, CapsLock → Ctrl, Right Alt → Ctrl, sticky keys
- Emacs: ~qvm-spice~ command and ~V~ keybinding in list buffer
- Emacs: ~qvm-keyboard~ command and ~k~ keybinding in list buffer
- ~VM_DISPLAY~ and ~VM_SPICE_PORT~ fields in ~vm.conf~

** Changed

- Emacs: list buffer columns updated from "VNC" to "Display" and "Port"
- ~qvm list~ and ~qvm info~ now show display type and corresponding port
- Bumped ~qvm.el~ version to 0.2.0

** Backward Compatibility

- Existing VMs without ~VM_DISPLAY~ in their config default to ~vnc~
- No changes required to existing VM configurations

* 0.1.0 — <2026-04-01>

Initial release.

** Added

- ~qvm~ shell script: Docker-inspired CLI for managing QEMU VMs
  - create, install, start, run, stop, ssh, scp, vnc, list, info, rename, rm
  - VNC display with auto-assigned ports
  - User-mode networking with SSH port forwarding
- ~qvm.el~ Emacs interface
  - Tabulated list buffer with keybindings for all operations
  - TRAMP integration for dired and eshell
  - Async command output buffer
  - VM creation from ISO with dired integration
